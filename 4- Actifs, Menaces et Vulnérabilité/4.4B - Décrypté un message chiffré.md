# Décryptage

## Scénario

Dans ce cas, tous les fichiers de votre répertoire personnel ont été chiffrés. Vous devrez utiliser des commandes Linux pour casser le chiffrement de César et déchiffrer les fichiers afin de pouvoir lire les messages cachés qu'ils contiennent.

Voici comment procéder : vous explorerez d’abord le contenu de votre répertoire personnel et lirez le contenu d’un fichier. Ensuite, vous trouverez un fichier caché et déchiffrerez le chiffrement de César qu’il contient. Enfin, vous déchiffrerez le fichier de données chiffré pour récupérer vos données et révéler le message caché.

OK, il est temps de décrypter quelques messages sous Linux !

> Cela commence lorsque vous êtes connecté en tant qu'utilisateur analyste, avec votre répertoire personnel, /home/analyst, comme répertoire de travail actuel.

## Solution 

1. Lire le contenu d'un fichier
* Utilisez la `ls` commande pour lister les fichiers du répertoire :
`ls /home/analyst`.

![chrome_5p8h7bnpSZ](https://github.com/Kwangsa19/Ketmanto-Cybersecurity-Portfolio/assets/135963482/2bf4701d-3283-4214-afaf-5d49ff8496ea)


* Listez le contenu du fichier README.txt : 
`cat README.txt`.

![chrome_OMbPE4mYh5](https://github.com/Kwangsa19/Ketmanto-Cybersecurity-Portfolio/assets/135963482/00d86c7f-a394-42f8-9f62-45522d3a5e91)

Le message contenu dans le fichier README.txt indique que le sous-répertoire caesar contient un fichier caché.

2. Trouver un fichier caché Dans cette tâche, vous devez trouver un fichier caché dans votre répertoire personnel et déchiffrer le chiffrement de César qu'il contient.
* Utilisez `cd` la commande pour accéder au sous-répertoire caesar et utilisez  `ls -a` pour lister tous les fichiers, y compris les fichiers cachés : `cd caesar` et `ls -a`.
  
![chrome_vVzk6li9Ir](https://github.com/Kwangsa19/Ketmanto-Cybersecurity-Portfolio/assets/135963482/a263ea12-a496-414a-b5da-9a4e87c956af)

* Utilisez la `cat` commande pour lister le contenu du fichier caché :
`cat .leftShift3`.

![chrome_2KE6gkd6Z3](https://github.com/Kwangsa19/Ketmanto-Cybersecurity-Portfolio/assets/135963482/35cddc8e-c1d6-4d09-886f-61a222559d9c)

Le message semble brouillé car il est chiffré avec un chiffrement de César. Ce chiffrement peut être déchiffré en décalant chaque caractère de l'alphabet vers la gauche ou vers la droite d'un nombre fixe de positions. Dans cet exemple, le décalage est de trois lettres vers la gauche. Ainsi `"d"` représente  `"a"`, et `"e"`représente  `"b"`.

* Déchiffrez le chiffrement à l'aide de la commande `cat` et `tr`:
`cat .leftShift3 | tr "d-za-cD-ZA-C" "a-zA-Z"`.

![chrome_yqTsKokU9s](https://github.com/Kwangsa19/Ketmanto-Cybersecurity-Portfolio/assets/135963482/2ae37e92-2b0d-460f-b1f0-7d31fb4921e5)

> La commande `tr` convertit un texte d'un ensemble de caractères à un autre, selon une correspondance. Le premier paramètre de `tr` représente l'ensemble de caractères d'entrée, et le second, l'ensemble de caractères de sortie. Ainsi, si vous fournissez les paramètres “abcd” et “pqrs”, et que la chaîne d'entrée de la commande `tr` est “ac”, la chaîne de sortie sera “pr".


4. Déchiffré un fichier 
* Retournez au répertoire de travail initial et exécutez cette commande pour déchiffrer un fichier :
`openssl aes-256-cbc -pbkdf2 -a -d -in Q1.encrypted -out Q1.recovered -k ettubrute`.

![chrome_vji0fyPapn](https://github.com/Kwangsa19/Ketmanto-Cybersecurity-Portfolio/assets/135963482/43b92274-8d80-44ba-b816-81dc13d25b4c)



Cette commande inverse le chiffrement du fichier à l'aide d'un chiffrement symétrique sécurisé, tel que AES-256-CBC. L' `-pbkdf2` Cette option permet d'ajouter une sécurité supplémentaire à la clé, et `-a` indique l'encodage souhaité pour la sortie.`-d` indique le déchiffrement, tandis que,`-in`--input` spécifie le fichier d'entrer et,`-out`--output` spécifie le fichier de sortie.`-k`--password` spécifie le mot de passe, qui est `ettubrute` dans cet exemple.

* Utilisez la `ls` commande et `cat Q1.recovered`.
